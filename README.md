# Airbnb Clone Project

This project is a full-stack clone of the Airbnb platform.

## Project Goals
- Build a functional accommodation booking web application
- Practice full-stack development concepts
- Apply responsive UI/UX design principles

## Tech Stack
- Frontend: HTML, CSS, JavaScript (React)
- Version Control: Git & GitHub
- Design: Figma
---

## UI/UX Design Planning

### Design Goals
- Create an intuitive and smooth booking flow
- Maintain visual consistency across the application
- Ensure fast loading and high performance
- Prioritize mobile-first and responsive design

### Key Features
- Property search and filtering
- Detailed property information view
- Secure and simple checkout process
- User authentication and session management

### Primary Pages

| Page Name | Description |
|----------|-------------|
| Property Listing View | Displays a grid of available properties with search and filter options |
| Listing Detailed View | Shows full property details including images, price, description, and booking form |
| Simple Checkout View | Handles booking confirmation and payment in a minimal and clear flow |

### Importance of User-Friendly Design
A user-friendly booking system reduces friction in the user journey and increases conversion rates.
Clear navigation, intuitive layouts, and responsive design help users complete bookings easily,
which improves user satisfaction and trust in the platform inspired by :contentReference[oaicite:0]{index=0}.

## UI/UX Design Specifications

### Color Styles
- Primary Color: #FF5A5F
- Secondary Color: #008489
- Background Color: #FFFFFF
- Primary Text Color: #222222
- Secondary Text Color: #717171

### Typography
- Primary Font Family: Circular
- Font Weights:
  - Book (400) – Secondary text
  - Medium (500) – Body text
  - Bold (700) – Headings
- Font Sizes:
  - Body Text: 16px
  - Secondary Text: 14px
  - Headings: 24px – 32px

### Importance of Identifying Design Properties
Identifying colors, typography, and layout styles from the mockup design in :contentReference[oaicite:1]{index=1}
ensures visual consistency across the application.
It helps developers implement the UI accurately and speeds up development by reducing design-related confusion.

## Project Roles and Responsibilities

| Role | Responsibilities |
|-----|------------------|
| Project Manager | Manages timelines, coordinates the team, and ensures project goals are met |
| Frontend Developers | Build UI components, implement responsive design, and connect APIs |
| Backend Developers | Develop APIs, manage the database, and handle business logic |
| Designers | Create UI mockups, maintain design consistency, and improve user experience |
| QA/Testers | Write test cases, test features, and report bugs |
| DevOps Engineers | Handle deployment, CI/CD pipelines, and server infrastructure |
| Product Owner | Defines requirements, prioritizes features, and represents stakeholders |
| Scrum Master | Facilitates agile practices, organizes meetings, and removes blockers |

Each role contributes to delivering a high-quality, scalable, and maintainable application.
